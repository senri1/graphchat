Next I’d tackle **official Codex interoperability**.

Build a Codex thread browser that:

1. Calls `thread/list`, grouped by working folder.
2. Opens a selected thread with `thread/read`.
3. Converts its linear turns into a GraphChat node chain.
4. Stores the real `threadId` and `turnId` checkpoints on imported nodes.
5. Lets us continue or branch it normally.
6. Verifies GraphChat-created threads appear in the official Codex app, and vice versa.
7. Refreshes imported chats without duplicating turns.

GraphChat folders should remain UI organization. Codex “projects” should be derived from each thread’s `cwd`, not mapped onto those folders.

The key acceptance test:

- Create a thread in the official Codex app.
- Open it in GraphChat.
- Continue it in GraphChat.
- See that continuation in the official app.
- Branch from an older node in GraphChat.
- See the fork as a separate Codex thread with the same working folder.

That would prove we genuinely have an alternate interoperable UI, rather than merely a separate app using the same runtime.
