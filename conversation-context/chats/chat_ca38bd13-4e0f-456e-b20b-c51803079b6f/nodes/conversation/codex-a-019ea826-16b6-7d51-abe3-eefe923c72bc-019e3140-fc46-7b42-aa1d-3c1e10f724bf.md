I checked the official OpenAI docs.

**Codex App Server** is OpenAI’s integration layer for embedding the Codex coding agent into your own client or product. It exposes the same Codex “harness” used by rich clients like the Codex VS Code extension through a JSON-RPC protocol. You run it with `codex app-server`, usually over `stdio`, with experimental WebSocket support. It gives a custom client access to Codex sessions, auth, conversation history, approvals, streamed events, skills, and rate-limit/account state. Source: [Codex App Server docs](https://developers.openai.com/codex/app-server), [OpenAI engineering post](https://openai.com/index/unlocking-the-codex-harness/).

**Sign in with ChatGPT for Codex** works through the app-server auth surface. Your client calls `account/login/start` with a ChatGPT login type, Codex returns either a browser `authUrl` with a localhost callback or a device-code flow, and after the user completes login the server emits `account/updated` with `authMode: "chatgpt"` and the user’s plan type. Codex then manages and refreshes the ChatGPT auth tokens. The app-server can also read ChatGPT Codex rate-limit buckets. Source: [Codex App Server auth endpoints](https://developers.openai.com/codex/app-server).

On the “can we use ChatGPT limits for custom apps?” part: **yes, but only in specific contexts.**

For **Codex**, yes: Codex is included in ChatGPT plans, and a custom Codex client built on App Server can use ChatGPT sign-in and the user’s Codex plan limits. OpenAI’s pricing docs say Codex is included in ChatGPT Free, Go, Plus, Pro, Business, Edu, and Enterprise plans, with limits depending on plan/model/task size; API-key mode is separate and charged at API rates. Source: [Codex pricing](https://developers.openai.com/codex/pricing), [Using Codex with your ChatGPT plan](https://help.openai.com/en/articles/11369540-using-codex-with-your-chatgpt-plan).

For **general custom applications**, no: a ChatGPT subscription is not a general-purpose OpenAI API allowance. If you build a normal standalone app with the Responses API/OpenAI API, you still pay API billing. The related “use ChatGPT limits” story is **ChatGPT Apps / Apps SDK**: you can build an MCP-backed app that runs inside ChatGPT, and ChatGPT usage follows the user’s normal ChatGPT plan limits. Your app server provides tools/data/UI; ChatGPT is the host model experience. Source: [Apps in ChatGPT](https://help.openai.com/en/articles/11487775-apps-in-chatgpt), [Apps SDK auth](https://developers.openai.com/apps-sdk/build/auth).

So the clean mental model is:

`Codex App Server` = build your own Codex client/harness and let users sign in with ChatGPT to use Codex limits.

`Apps SDK / MCP app in ChatGPT` = build an app that runs inside ChatGPT, where ChatGPT plan limits apply to the chat experience.

`OpenAI API app` = your API account pays; users’ ChatGPT Plus/Pro limits do not cover it.
