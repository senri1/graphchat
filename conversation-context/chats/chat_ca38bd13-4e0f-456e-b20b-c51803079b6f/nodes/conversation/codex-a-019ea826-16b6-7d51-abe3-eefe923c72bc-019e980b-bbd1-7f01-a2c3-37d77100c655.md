I’d do this in two controlled stages: first introduce the runtime abstraction without changing behavior, then add Codex App Server as the second runtime.

**1. Define The Runtime Contract**
Create `src/runtime/types.ts` with:

- Runtime identity and capabilities
- Authentication/account status
- Available models
- Generation requests
- Normalized streaming events
- Runtime-specific checkpoints
- Cancellation and approval requests

```ts
interface AgentRuntime {
  getStatus(): Promise<RuntimeStatus>;
  listModels(): Promise<RuntimeModel[]>;
  generate(request: RuntimeRequest): AsyncIterable<RuntimeEvent>;
  cancel(taskId: string): Promise<void>;
}
```

Codex-only operations such as native forking would be exposed as capabilities, not forced onto the Responses implementation.

**7. Add The Codex Runtime**

It will translate Codex notifications into common events such as:

```ts
{ type: 'approval-required', ... }

```

**8. Map Graph Branches To Codex Threads**
The behavior will be:

- First message in a branch: `thread/start` followed by `turn/start`
- Continue from the current branch tip: `turn/start` on its thread
- Branch from an earlier checkpoint: `thread/fork`, then `turn/start`
- Retry/regenerate: fork from the parent checkpoint
- Cancel: `turn/interrupt`
- Reopen: `thread/resume`

Every completed assistant node stores the resulting Codex checkpoint.

If a user edits history, GraphChat computes a `historyHash`. Descendant checkpoints with mismatched history become stale and are rebuilt from the closest valid ancestor rather than silently using incorrect Codex history.

**9. Project Folder Support**
Each GraphChat chat will optionally have a Codex working directory and execution policy:

```ts
{
  cwd: "/path/to/project",
  approvalPolicy: "...",
  sandbox: "..."
}
```

This controls the folder available to Codex’s file and command tools. Chats without a selected folder can run in a restricted GraphChat data/workspace directory.

**10. Preserve Sync And Mobile**
Google Drive continues syncing GraphChat’s portable graph, including Codex checkpoint IDs, but not:

- ChatGPT credentials
- Codex configuration
- Codex local thread logs
- Prompt caches

On another desktop, GraphChat attempts `thread/resume`. If unavailable, it rebuilds the branch from synced GraphChat messages.

Browser, Android, and iOS retain Responses API mode. Codex mode will initially be desktop-only because a mobile browser cannot spawn App Server.
